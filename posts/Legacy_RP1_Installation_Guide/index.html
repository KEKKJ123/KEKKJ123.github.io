二、安装前准备

再次提醒：在阅读此教程前，作者默认你会使用科学上网，若遇到网络问题请自行解决。

① 在 Steam 中下载 KSP（这里选择 1.12.5 版本），不要安装 DLC，找到 Kerbal Space Program 文件夹。

如果有其他的存档，建议备份。

最好将要安装 RP-1 的 KSP 文件夹迁移到其他位置，防止 Steam 意外修改文件。即使不安装模组也建议这样做。

② 在你的 KSP 根目录中加入 ckan.exe，部分用户可能没有后缀名显示 ckan，但你应当看到右边的文件类型显示应用程序。

③ 右键 ckan 创建快捷方式，对 ckan 的快捷方式右键，点击属性，在目标一栏的末尾输入“ –asroot”（前面有一个空格），点击确定。

④ 双击刚才的快捷方式打开 ckan，不出问题的话应该可以进入界面。如果提示报错，请你返回序重新看里面的文章。

⑤ 点击左上角的设置，点击兼容的游戏版本，勾选 1.12，点击确定。

经过了前面这么多的步骤，恭喜你可以着手安装 RP-1 Legacy Version 了。

三、正式安装和首次运行[2]

正式安装

① 找到并勾选 RP-1 Legacy Version。

② 找到并勾选 Realism Overhaul，在右边栏中点击版本，勾选 v15.0.1.0

③ 点击应用更改，点击右下角应用，点击左上角取消全选，再点击继续（不要乱选，除非你知道自己在安装什么模组）。

④ TweakScale Redistributable / TweakScale Rescaled Redistributable：两个均可用，它们在使用上并无差别。

⑤ KSPBurst / KSPBurst lite：请选择第一个，除非你真的差这 300 MB 流量。

⑥ Real Solar System Textures：数字越大的越清晰，但性能要求也越高。如果你不知道该选择哪个，4096 * 2048 即可。

⑦ 接下来进入安装，CKAN 会开始下载所需的模组，若提示模组 XXX 想要覆盖以下文件，选择是即可。

  • Test Flight / Test Lite：它们不是 RP-1 Legacy Version 的依赖，但仍建议勾选安装（不装还玩啥 RP-1 呢是吧）
  • Ven’s Stock Revamp / ReStock：它们都更改了原版的部件模型，在使用上并无差别，只是不能同时安装。

提示：由于 CKAN 本身机制，只有全部模组下载完毕后才会开始安装进程。若下载期间进度条卡住，可以点击取消，再多次重复上述步骤，直至全部下载完毕。

首次运行

① KSP 有画面后，你会看到两个弹窗，分别选择 Don’t show again 和是即可。

② 进入熟悉的画面后，会出现三个弹窗，一个个关闭即可。

  • 建议：为了获得较好的体验，请将渲染质量和贴图质量拉满，并打开抗锯齿，开启高亮特效，垂直同步请选择逐帧同步，其余设置根据个人喜好修改即可。
  • 若想屏蔽不兼容的 non RO / RP-1 部件（使用它们等于作弊），只需要在 Gamedata 中创建空文件夹 NoNonRO 和 NoNonRP0 即可。

如果加载中没有提示 Warning 和 Error，恭喜你成功安装了 RP-1 Legacy Version，坎星人迈出了一大步！

同时，为了更好的上手 RP-1，建议阅读官方常见问题[4]。

一些 Tips

这里是针对 RP-1 Legacy 的一些补充，其中的大多数并没有经过验证，在实践前请务必先备份你的存档。

  • RO Solar v1.2.0：在安装后，所有使用 1967+ 太阳能板的飞行中航天器的效率都将会降低一个科技等级。
  • Dynamic Battery Storage：直接删除整个文件夹即可。它是 Near Future Electrical 的依赖，但基本无用（至少在 Legacy 是这样），因为有关电量的统计功能可以使用 Kerbalism。
  • Kerbal Changelog：同上。它的作用是显示模组的更新日志（包括它自己），但在已经停止更新的 Legacy 上显然没什么用处。顺带一提，RP-1 的 Express Install 列表已经没有这个模组了，所以完全没有安装的必要。
  • Toolbar 和 WASD Editor Camera 一起使用存在兼容性问题，导致建造界面的内存占用持续升高，以及额外的 CPU 和磁盘占用。但仍需进一步测试。详见问题报告。已修复，请更新 Toolbar Controller 至 v0.1.9.12+ 版本。
  • Kerbal Engineer Redux：存在 Δv 误报的情况，补丁的说明见此。
  • MechJeb Dev：提供了一些更加“定制化”的功能，可选。如果要使用请务必阅读官方说明。
  • ProbesPlus：由于太过陈旧，部件参数已经不再适合定制化探测任务，但如果只用于演示和拍摄则非常合适。
  • Navball Docking Alignment Indicator CE-2：MJ 的智能姿态控制拥有指向的功能，但此模组提供了额外的可视化标记。
  • ROLoadingImages：原生支持 RP-1。安装后需要手动将 ROLoadingImages\PluginData\Screens 与 RP-0\PluginData\Screens 合并，然后删除 ROLoadingImages 文件夹即可。
  • VAB Organizer-RP-1 Configs：原生支持 RP-1。尽管在 Legacy RP-1 并不能完全生效，但依然很有用。
  • Waterfall Core：右侧工具栏 WaterfallFX editor 的图标可依照此文章隐藏。（来自 Bilibili @逗比去哪2）

四、拓展模组列表（持续更新）[1]

成功启动后，尽管游戏已经拥有了 RP-1 的全部内容，但它仍然很简陋。

这里推荐安装一些拓展模组，使游戏内容更丰满。有的提供了额外的引擎，有的增加了实用的功能，有的改善了视觉效果。

带 $ 为作者正在使用的模组，仅代表个人习惯。

此处所列模组均可通过 Bing 搜索引擎以 KSP＋模组名字的方式检索到。

标注版本号的模组需安装对应版本，否则会无法安装、报错或无法达到预期效果。

若未标注则安装最新版，报错请在下方评论，作者会更新相关条目。

$ Near Future Exploration - 添加了许多天线。

Near Future Launch Vehicles - 虽然没有完全配置，但提供了许多现代引擎，如 RD-701/704、AR-1、RS-84 和 STBE-1，以及 TR-107。

2. VAB / SPH 建造工具类

$ Custom Parachute Message - 高度自由化的可定制降落伞配色。

$ Editor Extensions Redux - 添加了更多的对称选项、更精细的角度限制等功能。

Hangar Grid Regridded - 添加视觉上的“激光”网格和自动对准工具来简化设计。

$ Kronal Vessel Viewer（需要手动勾选）- 已过时，但仍在 1.12 可用。可以生成美观清晰的正交投影设计图。

$ Kerbal Engineer Redux - 可以显示许多有用的信息。DeltaV 计算有误差，详见补充。

$ KSP Craft Organizer Improved - 重写了原版的航天器文件系统，并添加了标签等实用功能。

$ MechJeb 2 - 大名鼎鼎的 MJ，全能型工具无需多言。

$ Moar Filter Extension Configs - 向部件列表添加基于直径的分类，其中还有子类别。尺寸过滤的范围为 0.35 - 30 米。

RCS Build Aid - 建造火箭时，用于平衡 RCS 和引擎围绕质心的辅助工具。

Real Time Clock 2 - 新增了一个显示现实世界时间的小窗口。妈妈再也不用担心我造火箭会忘记吃饭了

$ ROFilterExtensionConfig - 适用 RO/RP-1 的 Filter Extensions 配置。

$ Toolbar - 配合 Toolbar Controller 可以精简原版中繁杂的工具栏。

$ VAB Organizer - 修改了部件的排列模式，主要有部件尺寸标签、子分类和高级分类筛选自定义三个分类功能。

$ VAB Organizer-RP-1 Configs（需要手动安装）- 添加了以国家分类的引擎标签。详见补充。

$ WASD Editor Camera - 提供了另一种用 WASD 控制视角的方法，可以使设计航天器更加容易。

Werner Checklist Continued - 向 VAB 添加一个检查表，用于在发射前检测某些重要部件是否安装。还添加了一个飞行前检查表，可以在飞行界面访问该清单。有助于防止犯错，不得不回退到 VAB。

3.飞行界面类

ABookCase Orbital Reference System - 将鼠标悬停在地图视图中当前飞行器的轨道上，即可查看时间和高度信息。显而易见的不支持 Principia。

$ AltimeterAutoHide - 一个很简单的模组，让高度计可以自动固定和隐藏。

$ BetterBurnTime - 具有燃烧时间、撞击时间、交会时间等许多信息的显示，非常实用。

$ BetterTimeWarpContinued - 自定义正常和物理加速倍率。

Chatterer & Chatterer Extended - 加入了科学仪器、数据传输和无线电的声音，还增加了一些环境声音。

$ Critical Temperature Gauge - 显示到达临界值的部件温度，用于监控航天器热量系统。

Docking Port Alignment Indicator - 添加了一个显示对接相关参数的可视化窗口。

$ Easy Vessel Switch - 方便且平滑地聚焦到某个部件，或切换到其他的近距离航天器。

$ Historian Expanded - 在屏幕截图中添加可配置的集成 HUD / 信息栏。

HyperEdit（需要手动勾选） - 已过时，但仍在 1.12 可用。一个可修改轨道参数的“作弊”模组，可用于测试各种航天器。（Principia 环境需开启 Debug 使用）

$ Kerbal Alarm Clock - 闹钟，用于提示任务开始。几乎是深空长程任务的必需品。

$ KSC Switcher - 允许更改 KSC 的地理位置，提供了全球不同地区的几十个发射台。

KSP Resonant Orbit Calculator - 兼容 RSS 的共振轨道的可视化计算工具，与 KAC 联动。

Lunar Transfer Planner - 大大简化了规划月球发射任务的过程，非常适合在 Principia 环境使用。

$ NavHud - 全屏导航球覆盖。让您知道你的航天器指向何处。

Navball Docking Alignment Indicator CE-2 - 在导航球上添加了新的指示方向，即目标朝向。详见补充。

Precise Node - 添加机动节点的 GUI，允许更精确的调整节点，精度比较高。与 Principia 不兼容。

Precise Maneuver - 同上。理论上不兼容 Principia，未测试。

$ RSS DateTime Formatter - 用正常的日历形式代替原版时间的表示方法。

$ Texture Replacer（建议手动安装） - 允许更改坎巴拉人的皮肤和航天服纹理。自定义星空背景也依靠此模组运行。ckan 安装可能会有bug。

Time Control - 另一个调整时间加速倍率的模组。功能相较于 BetterTimeWarpContinued 更多。

$ Trajectories - 落点轨迹显示，在有大气环境会根据阻力动态计算，较为准确。

$ Transfer Window Planner - Fork - 行星际转移窗口规划，可以非常方便的算出相关参数，同样适用 Principia 环境。

Waypoint Manager - 显示飞行中的路径点，并提供了自定义路径点。对着陆到指定地点的情况非常合适。

4. 修复 / 机制类

  • 此类别模组除 Chinese Mod Patches、kOS: Scriptable Autopilot System 和 Principia 这三个外，建议所有玩家安装。

$ Adjustable Mod Panel - 允许你禁用原版工具栏的模组按钮。

$ Chinese Mod Patches - 一个 KSP 社区 Mod 的中文翻译补丁合集（作者自述）。

$ Deferred（建议阅读官方说明）- 添加了延迟渲染支持，允许使用更现代的照明技术。渲染更逼真的金属纹理、环境光遮蔽和原生的行星反射光。建议主动勾选安装。

$ Keep It Straight - 禁用不同情况下的视角自动切换，转而记忆以往的视角。

kOS: Scriptable Autopilot System - 可编程自动驾驶系统。强烈建议阅读官方介绍食用。

$ KSP Wheel - 修复在大尺寸星系下，原版起落架和轮子的碰撞箱问题，并提供了更多的调试选项。一般是其他模组的依赖，但仍建议主动勾选安装。

$ Principia（新手不建议安装） - 多体引力模组。具体介绍请阅读官方wiki 或 Part 1。

$ QuickExit - 在任何场景都能快速方便的退出 KSP。

$ QuickGoTo - 在任何场景都能快速的切换到其他场景，如 VAB、跟踪站、任务中心等。

$ S.A.V.E Resaved - 自动备份系统，用于备份和恢复存档。不是快速保存（QuickSave）。

$ Stock Alarm Clock Disabler - 移除了原版自带的闹钟功能。建议与 Kerbal Alarm Clock 配合食用。

$ Tracking Station Evolved - 添加了依照所在天体、发射时间、航天器类型等的分类方式。

$ Zero MiniAVC - 自动删除各个模组自带的 MiniAVC。

5. 美化类（请适度安装）

▶
RSS-Origin：一个太阳系拓展模组，添加了大量天体，以及提供了更好的纹理、星环、星空背景等。
  • 安装前请务必阅读 RSS-Origin 模组介绍（P1）和安装指导（P2）！

$ RSS-Origin Core - RSS-Origin 的核心依赖。添加占星学符号，修正 RSS 天体的轨道（适用 Principia 环境）等一系列功能。

RSS-Origin CelestialsPack（在 RP-1 环境不建议与 Principia 同时安装） - 在原 RSS 的基础上增加了上百个的小行星、彗星、卫星、矮行星等星际天体。

RSS-Origin RSSTexture&TopoRevamp - 使用更准确、更真实和更高分辨率的纹理和地形对一些 RSS 天体进行了修改。

$ RSS-Origin JSUNrings - 为太阳系中的四颗气巨行星添加了准确逼真的行星环。

$ RSS-Origin GalaxyTex - 高质量的星空背景，有 8k / 16k / 32k / 64k 四种不同的分辨率可选。

$ Distant Object Enhancement /L - 提供了远距离天体和航天器的视觉效果，几乎不影响性能。

Docking Port Sound FX - 添加了对接时的碰撞声音效果。

$ Firefly - 重写了气动和再入效果。安装后请关闭原版自带的气动效果。

KSP2 Pre-Alpha Style NavBall - KSP2 风格的导航球。

Milkyway Skybox - SpaceEngine - 作者使用的上一个星空背景。

$ Parallax - 对原版地形着色器的重写。尽管在 RSS 无法提供和原版一样颠覆的效果，但有效解决了着陆时地形贴图的“马赛克”。

PlanetShine - 添加了行星自身的反射光照，且几乎不影响性能。在默认配置下与 Deferred 同时安装会无法生效。

Probe Control Room Recontrolled - 添加了虚假的航天中心操控航天器的 IVA 画面。

Reentry Particle Effect Renewed - 添加了一个原版存在但未使用的再入特效。

$ ROLoadingImages（需要手动安装） - 添加了 2022 年夏季和秋季比赛的一些截图画面。详见补充。

$ RSS Visual Enhancements - 一个经典的支持真实太阳系的美化包，有高低两种分辨率可选。

$ Scatterer - 添加了逼真的大气散射、水面及水下特效和阳光渲染等各种效果。一般是其他模组的依赖，但仍建议主动勾选安装。

TUFX - 提供画面的后处理效果。建议额外安装配置文件使用。

$ Waterfall Core - 引擎尾焰美化。详见补充。

$ ZTheme - 将原版主题替换成深色主题。

6. 黑名单模组（不建议、不兼容或没有配置）

注意：对于无配置的模组，可以手动撰写 patch 文件修正。

Infernal RO-Robotics（默认无配置） - 添加了适用 RO 的机械转轴。TD Channel 曾在他的视频中使用过，配置文件来源于 Reddit。

Kerbal Planetary Base Systems（无配置） - 老牌基地组件系统。

Shadow Space Technologies Unlimited （不兼容） - 在任何情况下都不应在 RP-1 中使用 SSTU。可使用 ROTanks 和 Procedural Parts 替代相关的功能。

Stage Recovery / Flight Manager for Reusable Stages（不兼容） - 会破坏游戏早期的平衡，并且会向 log 中发送垃圾消息，降低游戏性能。

Textures Unlimited Recolor Depot（不建议） - 在 RP-1 中基本上没用，因为您将主要使用 Ven 或 SXT 的部分，而这个模组不会触及这些部分。如果您使用 ReStock，则完全没用。不值得花费额外的时间去加载。

KRASH / Kerbal Construction Time（不兼容） - 模拟现在由 RP-1 内置的 KCT 处理。

Engine Lighting Relit（不兼容） - 没有 RO 配置，并且会向 log 中发送垃圾消息。

Deadly Reentry（不兼容） - 可能会破坏游戏物理效果。RO Heatshields 完全取代了它。

以及其他未提到的部件类模组。

7. 待测试

这里是作者存留待测试模组的备忘录，在这里的模组不受支持。

如果有好心人帮忙测试就再好不过了

DiRT - 已过时，最高支持 1.10。是 Texture Replacer 的一个分支，目的是简化插件，删除除纹理替换之外的所有内容。

Texture Replacer - 样本量较少，需要更多的测试手动安装的情况。

SCANsat - 其 v20.x 版本在与 RSS 一起使用时会消耗大量内存（通常超过 32 GB）。如果要使用，则应通过转到 Gamedata/SCANsat/PluginData/Settings.cfg 并设置 VisibleMapsActive 到 false 来禁用视觉地图。

$ Rocket Sound Enhancement - 无 RO 配置，可能与 ROWaterfall 冲突发生 MM 报错。但在 Legacy 并未发生上述情况。

RNMisc（建议手动安装）- 对一些杂项部件的纹理重制。暂时不清楚其依赖源。

Σκοπός - 仅指定大框架目标，将具体任务设计留给玩家的 KSP 合同系统。目前仅适配 RP-1 的通信网络合同。需要进一步测试兼容性。

Precise Maneuver - 理论上不兼容 Principia，未测试。

Part Commander Continued - 待测试。

Parallax Continued - 不支持 RSS，但支持 Sol，参见支持的行星模组列表。对 RP-1（包括 Legacy）的兼容性未知。
