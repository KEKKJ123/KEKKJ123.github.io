一、基本机器人、主控模块和电源管理模块（简称电管模块）

1.基本机器人

如果不考虑离线模块，最基本的机器人由三部分组成：电源、电源管理模块和主控模块。

基本机器人便于加装和调试其他模块。若无特殊说明，下文所指机器人均为基本机器人。

如何组装？按照电源——电管模块——主控模块的顺序串联即可。

2.主控模块

  • 部分主控模块到手为英文模式。

    解决方法：长按确定键进入菜单，移动光标至 Language，点击确定后再选择简体中文，确定后即可恢复中文。

  • 主控模块需要在已连接 PC 的情况下安装驱动，否则可能无法正确识别。下载链接[7]在此。

  • 主控必须连接电管模块的黑色航空线接口，否则无效。其余接口无顺序要求。

二、选手端、裁判端和服务器的搭建

  • 注意：选手端、裁判端和服务器的文件路径不要有任何中文字符！！

1.裁判端和服务器

自25赛季开始，服务器需要连接互联网（大疆账号），这显著增加了搭建服务器的繁琐程度。

搭建服务器前请务必关闭本机网络防火墙！

这里贴一个链接，RM-Referee-Aid：RM裁判系统工具箱，内含关闭防火墙的功能。

只有 RM Server 的日志系统窗口标题为 [Online] 时，服务器才算搭建成功！

双网卡方案（无需路由器，但概率失效）

由于作者所在实验室的网络无法连接到互联网，故摸索了一个基于双网卡的搭建方案。

  • 使用双网卡方案时，科学上网工具的 TUN 模式无法使用。

经过测试，本方法适用连接校园网的情况。但样本数极少，无法保证可靠性，若不成功可使用个人热点等联网方式替代。

  • 搭建方法

① 主机连接 WiFi（有线网未测试）并能够正常连接互联网。

② 打开主机的个人热点（与其他设备共享我的 Internet 连接），选择 2.4G 频带。

③ 打开控制面板，依次点击网络和共享中心，更改适配器设置，Microsoft Wi-Fi Direct Virtual Adapter，属性。

④ 双击 Internet 协议版本 4（TCP/IPv4），更改 IP 地址为 192.168.1.2，子网掩码为 255.255.255.0，一路点击确定。

此时就能够保证服务器能够连接互联网，且其他机器人可连接到本机共享的移动热点。

【待验证】路由器方案（建议使用较好的路由器）

单主机双网卡方案虽简单，但比较依赖电脑自带网卡的性能，有概率无法搭建成功（服务器连接丢失）。

若有额外的路由器则能大大减少出错概率。

三、激活图传模块，搭建图传链路

RoboMaster Tool 2 是裁判系统固件版本的管理工具。它还能激活图传模块。下载链接[2]在此。

建议使用最新版本的 RM Tool、裁判端、选手端和模块固件！

0.连接主控，查询和更新固件版本

① 搭建基本机器人，不要上电，用 Micro-USB 数据线连接主控和 PC 后再打开电源。

② 打开 RM Tool，显示网络正常后再选择并打开串口，此时应当显示绿色的对勾标志。

  • 若不成功，可尝试重新插拔 Micro-USB 数据线。

③ 在已连接基本机器人的前提下，需要升级固件版本的模块会显示红色。

④ 点击升级。升级期间，不要断电或操作模块。

1.图传模块 VT02（发送端）的激活

① 在已执行第零节的前提下，将发送端与基本机器人连接，检查模块版本是否需要升级。

② 打开串口，待出现绿色对勾标志后，点击图传激活，点击发送端激活。

③ 确认激活区域为中国和其他地区后，再点击激活。

  • 若显示激活区域为日本地区，请检查 PC 的网络环境。
  • 必要时可以科学上网开启香港节点，不要使用台湾节点。（From 视觉救命群群友）

2.图传模块 VT12（接收端）的激活

① 使用双头 Type-C 数据线连接 PC 和接收端，用适配器给接收端供电。

② 打开 RM Tool，显示网络正常后再选择并打开串口，此时应当显示绿色的对勾标志。

③ 安装接收端驱动。下载链接[5][6]在此。

④ 点击图传激活，点击接收端激活。在弹出的窗口中再次打开对应的串口。

⑤ 确认激活区域为中国和其他地区后，再点击激活。注意事项见上。

3.图传链路的搭建

  • 发送端

连接发送端与基本机器人，并使用主控更改机器人 ID 和类型。

  • 接收端

① 使用双头 Type-C 数据线连接 PC 和接收端，用适配器给接收端供电。

② 打开 PC 的设备管理器，双击网络适配器，检查有无 Remote NDIS based Iinternet Sharing Device。

③ 根据前一节内容，安装接收端的驱动。

④ 打开选手端，进入客户端后按 P 键打开设置面板，登录发送端机器人的 ID。

⑤ 待图传串口和连接状态都变为绿色后，若未出现图传画面，请断电重启接收端。

  • 若半分钟后指示灯依然为红色，请重启发送端。
  • 若重启后依然无法连接，请参考第二章第一节的搭建方法，找到 Remote NDIS based Iinternet Sharing Device，检查 IP 地址是否为 192.168.42.105，并再次重启接收端。
